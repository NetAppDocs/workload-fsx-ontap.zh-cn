. *FSx for ONTAP file system*：为目标FSx for ONTAP文件系统选择ONTAP文件系统的凭据、区域和FSx名称。
. *Storage VM name*：选择 Storage VM。您选择的 Storage VM 是此复制关系中所有选定卷的目标。
+
当源和目标存储虚拟机名称相同时，我们会为目标存储虚拟机名称创建一个文件系统别名，以避免对等冲突。

. *卷名*：目标卷名称将自动生成，格式如下 `{OriginalVolumeName}_copy`。您可以使用自动生成的卷名称或输入其他卷名称。
. *分层策略*：为目标卷中存储的数据选择分层策略。对于您选择的使用情形、此层策略默认为建议的层策略。
+
使用 Workload Factory 控制台创建卷时，_平衡（自动）_ 是默认分层策略。有关卷分层策略的更多信息，请参阅link:https://docs.aws.amazon.com/fsx/latest/ONTAPGuide/volume-storage-capacity.html#data-tiering-policy["卷存储容量"^]在 AWS FSx for NetApp ONTAP文档中。请注意，工作负载工厂在工作负载工厂控制台中使用基于用例的名称来制定分层策略，并在括号中包含 FSx for ONTAP分层策略名称。

+
如果您选择了迁移用例，Workload Factory 会自动选择将源卷的分层策略复制到目标卷。您可以取消选择复制分层策略并选择适用于选择进行复制的卷的分层策略。

. *最大传输速率*：选择*受限*并输入最大传输速率(MB/秒)或者，选择*Unlimited (无限制)*。
+
如果没有限制、网络和应用程序性能可能会下降。或者、对于关键工作负载(例如、主要用于灾难恢复的工作负载)、我们建议对用于ONTAP文件系统的FSx使用无限传输速率。。在复制设置下、提供以下内容：

. *初始化复制关系*：选择在创建复制关系后立即初始化复制关系，或稍后再初始化。请注意，初始化可能很耗时，您可能希望在非高峰时段运行基线传输。
+
要稍后手动初始化，请参阅 <<步骤 2：初始化复制关系,步骤 2：初始化复制关系>> 中的说明。

. *复制间隔*：选择从源卷向目标卷传输快照的频率。
. *长期保留*：(可选)启用快照以进行长期保留。长期保留使业务服务即使在站点完全瘫痪时也能继续运行、从而支持应用程序使用二级副本透明地进行故障转移。
+
未启用长期保留的复制将使用 _MirrorAllSnapshots_ 策略。启用长期保留会将 _MirrorAndVault_ 策略分配给复制。

+
如果启用长期保留、请选择现有策略或创建新策略以定义要复制的快照以及要保留的数量。

+

NOTE: 长期保留需要匹配的源标签和目标标签。如果需要，Workload Factory 可以为您创建缺少的标签。

+
** *Choose an existing policy*：选择现有策略。
** *创建新策略*：输入*策略名称*。


. *不可变快照*：可选。选择*启用不可更改的快照*以防止在保留期间删除此策略中创建的快照。
+
** 以小时、天、月或年为单位设置*保留期限*。
** *Snapshot策略*：在表中，选择快照策略频率和要保留的副本数。您可以选择多个快照策略。


. *S3 访问点*：可选地，通过 AWS S3 API 连接 S3 访问点，以访问驻留在 NFS 或 SMB/CIFS 卷上的 FSx for ONTAP文件系统数据。仅支持文件访问类型。提供以下详细信息：
+
** *S3接入点名称*：输入S3接入点的名称。
** *用户*：选择有权访问该卷的现有用户或创建新用户。
** *用户类型*：选择*UNIX*或*Windows*作为用户类型。
** *网络配置*：选择 *Internet* 或 *Virtual private cloud (VPC)*。您选择的网络类型决定了接入点是可从互联网访问还是仅限于特定的 VPC。
** *启用库存表*：在卷上启用库存表时，系统会为 S3 访问点可访问的所有对象生成元数据，并产生 AWS S3 请求成本。有关详细信息，请参见 link:https://aws.amazon.com/s3/pricing/["Amazon S3 定价文档"^]。


. *S3 接入点标签*：您最多可以添加 50 个标签（可选）。
